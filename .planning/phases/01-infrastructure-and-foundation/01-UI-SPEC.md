# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Infrastructure and Foundation.
> This phase delivers no rendered user interface. Its sole UI output is the design token system in `globals.css` and the Cascadia Code font pipeline that all subsequent phases inherit.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | none |

## Spacing Scale

Token naming convention: CSS custom properties on `:root` in `globals.css`. Variable names use the `--space-N` pattern matching the 8px grid declared in CONTEXT.md D-05.

| Token | Value | Usage |
|-------|-------|-------|
| `--space-1` | 8px | Tight icon gaps, inline code padding |
| `--space-2` | 16px | Default element spacing, button padding |
| `--space-3` | 24px | Card internal padding |
| `--space-4` | 32px | Section padding, layout gaps |
| `--space-6` | 48px | Major section breaks |
| `--space-8` | 64px | Page-level vertical rhythm |
| `--space-10` | 80px | Nav bar height (matches existing `height: 80px`) |
| `--space-16` | 128px | Top-of-page hero clearance |

Source: CONTEXT.md D-05, D-06. Intermediate values (--space-2 through --space-16) are at Claude's discretion per CONTEXT.md.

Exceptions: `--space-half: 4px` — reserved for sub-pixel nudges (icon alignment, border offsets). Must not be used as general spacing.

## Typography

Font family token: `--font-mono` — set to the CSS variable emitted by `next/font/local` at `app/layout.tsx`, falling back to `Consolas, 'Courier New', monospace`. Source: CONTEXT.md D-12, D-13, D-14.

All sizes are defined as CSS custom properties on `:root`. Downstream phases reference tokens, never hardcoded values.

| Role | Token | Size | Weight | Line Height |
|------|-------|------|--------|-------------|
| Body | `--text-base` | 16px | 400 (regular) | 1.5 |
| Label / small | `--text-sm` | 14px | 400 (regular) | 1.4 |
| Heading | `--text-lg` | 20px | 600 (semibold) | 1.2 |
| Display | `--text-xl` | 32px | 600 (semibold) | 1.1 |

**Weights declared:** 400 (regular) and 600 (semibold) only. No other weights are permitted.

**Source rationale:** Existing `styles.css` uses a range of 0.85rem–3rem (13.6px–48px) across the whole site. Phase 1 establishes the normalized token set that the Next.js migration will use — the 3rem display is an outlier used only in the hero, which Phase 3 will address. The four tokens above cover all typographic roles without fragmentation.

## Color

All values defined as CSS custom properties on `:root` in `globals.css`. Source: CONTEXT.md D-01 through D-04, D-07 through D-09.

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `--color-bg` | `#0a0a0a` | Page background, card backgrounds |
| Secondary (30%) | `--color-surface` | `#111111` | Elevated surfaces, nav backdrop, overlay |
| Accent primary (8%) | `--color-teal` | `#00ffcc` | Primary emphasis, teal borders, focus rings, active nav indicators |
| Accent secondary (2%) | `--color-blue` | `#7dd3fc` | Secondary elements, links, subdued highlights |
| Destructive | `--color-error` | `#ff4d4d` | Destructive actions only — not used in Phase 1 |

**Accent reserved for (teal `#00ffcc`):**
- Focus-visible outline rings (keyboard navigation)
- Active/current nav indicator
- Card border glow
- Primary CTA button border

**Accent reserved for (blue `#7dd3fc`):**
- Inline hyperlinks
- Secondary metadata text (tech stack labels, timestamps)

**Opacity variants** — also defined as tokens, not inline `rgba()` values:

| Token | Value | Usage |
|-------|-------|-------|
| `--color-teal-subtle` | `rgba(0, 255, 204, 0.15)` | Hover state backgrounds |
| `--color-teal-glow` | `rgba(0, 255, 204, 0.20)` | Glow shadow component |
| `--color-surface-nav` | `rgba(10, 10, 10, 0.95)` | Nav backdrop with blur |

Source: CONTEXT.md D-03, D-04; `styles.css` existing values `rgba(0, 255, 204, 0.15)` and `rgba(10, 10, 10, 0.95)`.

## Border and Glow Tokens

These are design tokens, not interaction patterns. Defined in Phase 1, consumed from Phase 2 onward.

| Token | Value | Usage |
|-------|-------|-------|
| `--glow-teal` | `0 0 8px rgba(0, 255, 204, 0.20)` | Terminal card ambient glow |
| `--glow-teal-nav` | `0 0 20px rgba(0, 255, 204, 0.15)` | Nav bar bottom glow |
| `--radius-card` | `4px` | Card border radius — minimal, maintains terminal sharpness |

Source: CONTEXT.md D-07, D-08, D-09.

**No hard borders on cards.** Cards use `box-shadow` glow only. Source: CONTEXT.md D-07.
