Add validation function (fires on submit only, per D-10):
```tsx
function validate(form: FormData): { name?: string; email?: string; message?: string } {
  const errs: { name?: string; email?: string; message?: string } = {}
  if (!form.get('name')?.toString().trim()) errs.name = 'Name is required'
  const email = form.get('email')?.toString().trim() ?? ''
  if (!email) errs.email = 'Email is required'
  else if (!/\S+@\S+\.\S+/.test(email)) errs.email = 'Valid email required'
  if (!form.get('message')?.toString().trim()) errs.message = 'Message is required'
  return errs
}
```

Add submit handler (per D-01, D-13):
```tsx
async function handleSubmit(e: FormEvent<HTMLFormElement>) {
  e.preventDefault()
  const form = new FormData(e.currentTarget)

  // Honeypot check — silently discard bot submissions (per D-13)
  if (form.get('bot-field')) {
    setStatus('success')
    return
  }

  // Validate (per D-09, D-10, D-11, D-12)
  const validationErrors = validate(form)
  setErrors(validationErrors)
  if (Object.keys(validationErrors).length > 0) return

  setStatus('submitting')

  try {
    const response = await fetch('/', {
      method: 'POST',
      headers: { 'Content-Type': 'application/x-www-form-urlencoded' },
      body: new URLSearchParams(form as unknown as Record<string, string>).toString(),
    })
    if (!response.ok) throw new Error('Submission failed')
    setStatus('success')
  } catch {
    setStatus('error')
  }
}
```

Form JSX — remove `action="/"` attribute (per D-03), add `onSubmit={handleSubmit}`, add `netlify-honeypot="bot-field"`:
```tsx
<form
  className={styles.form}
  name="contact"
  data-netlify="true"
  netlify-honeypot="bot-field"
  method="POST"
  onSubmit={handleSubmit}
>

    <label>Don't fill this out: <input name="bot-field" /></label>

```

Each field gets an error message below it (per D-11). Use `aria-describedby` linking input to error for accessibility:
```tsx
<div className={styles.field}>
  <label htmlFor="name" className={styles.label}>Name</label>
  <input
    type="text"
    id="name"
    name="name"
    className={styles.input}
    aria-describedby={errors.name ? 'name-error' : undefined}
    aria-invalid={errors.name ? true : undefined}
  />
  {errors.name && <span id="name-error" className={styles.fieldError}>{errors.name}</span>}
</div>
```

Same pattern for email (type="email") and message (textarea with rows={5}).

Submit button changes text based on status (per D-06):
```tsx
<button type="submit" className={styles.submit} disabled={status === 'submitting'}>
  {status === 'submitting' ? '[ SENDING... ]' : '[ SEND MESSAGE ]'}
</button>
```

Conditionally render success state (per D-05) or error state (per D-07) instead of the form when status is 'success' or 'error':
```tsx
{status === 'success' ? (
  <p className={styles.success}>[ MESSAGE SENT ] — I'll be in touch.</p>
) : status === 'error' ? (
  <>
    <p className={styles.errorMessage}>[ ERROR ] — Something went wrong. Try again or email me directly.</p>
    {/* Render the form below so user can retry */}
    <form ...>...</form>
  </>
) : (
  <form ...>...</form>
)}
```

Note on error state: When status is 'error', the error message displays above the form, and the form remains interactive so the user can retry (per D-07). Reset status to 'idle' is not needed — the form is still rendered.
