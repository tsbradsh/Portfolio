Create `app/components/Nav.tsx` as a `'use client'` component with the following structure:

**State management:**
- `const [isOpen, setIsOpen] = useState(false)` — hamburger nav state
- `const [dropOpen, setDropOpen] = useState(false)` — desktop Projects dropdown state
- `const hamburgerRef = useRef<HTMLButtonElement>(null)` — for focus return on Escape
- `const dropdownRef = useRef<HTMLDivElement>(null)` — for click-outside detection

**Import navLinks and projects from `app/data/projects.ts`:**
- `import { navLinks, projects } from '../data/projects'`

**Nav structure (per UI-SPEC Section 5 + Section 10):**
```
<nav className={styles.nav}>
  <!-- Logo: <img src="/assets/1x/logocolor.png" alt="Tyler Bradshaw logo" height={70} /> -->
  <!-- Wordmark: <img src="/assets/1x/wordmark.png" alt="Tyler Bradshaw" height={50} /> -->
  <!-- Hamburger <button> with aria-label, aria-expanded, aria-controls="links" -->
  <!-- 3 <span> bars that animate to X when open -->
  <!-- Overlay: {isOpen && <div className={styles.overlay} onClick={close} aria-hidden="true" />} -->
  <!-- <ul id="links"> with nav items -->
</nav>
```

**Nav items (desktop):**
1. `INFO` — `<a href="#info">`
2. `SKILLS` — `<a href="#skills">`
3. `PROJECTS` — desktop: caret `<button>` triggers dropdown with 3 project links; mobile: 3 direct links (D-07)
4. `CONNECT` — `<a href="#connect">`
5. `[ GITHUB ]` — `<a href={navLinks.github} target="_blank" rel="noopener noreferrer">`
6. `[ LINKEDIN ]` — `<a href={navLinks.linkedin} target="_blank" rel="noopener noreferrer">`

**Desktop Projects dropdown:**
- Wrapper `<div ref={dropdownRef}>` containing caret `<button>` and submenu `<div>`
- Caret button: native `<button>` element (NOT svg role="button"), with `aria-expanded={dropOpen}`, `aria-haspopup="true"`, text content is a down-arrow character `▾`
- Submenu contains 3 `<a>` links: one per project from `projects` array, href `#{project.id}`
- Click outside closes dropdown via `useEffect` + `document.addEventListener('mousedown')` checking `dropdownRef.current.contains(e.target as Node)`
- Submenu hidden via CSS `display: none` / `display: block` toggled by className

**Mobile nav (at <= 768px via CSS media query):**
- Hamburger visible, `<ul id="links">` hidden by default
- When `isOpen`, links list shown as full-width fixed overlay drawer
- Projects dropdown flattened: dropdown wrapper hidden, 3 individual project links shown as direct `<li>` items at same level as other nav items (D-07)
- Use separate mobile-only `<li>` elements for each project that are `display: none` on desktop and `display: list-item` at `<=768px`
- Every `<a>` and project link has `onClick={() => setIsOpen(false)}` for auto-close (D-06)

**Keyboard behavior:**
- Escape key closes nav: `useEffect` adds `keydown` listener when `isOpen`, calls `setIsOpen(false)` then `hamburgerRef.current?.focus()` (D-05, research open question 1)
- Body scroll lock: `useEffect` toggles `document.body.style.overflow = isOpen ? 'hidden' : ''`, cleanup restores `''`

**Hamburger animation:**
- 3 `<span>` elements with CSS classes toggled based on `isOpen`
- Open state: top bar rotates 45deg + translateY(6px), middle bar opacity 0, bottom bar rotates -45deg + translateY(-6px)
- Transition: `200ms ease` on transform and opacity
